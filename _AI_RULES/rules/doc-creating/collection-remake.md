# Режим Plan Mode
Создаёт подробные планы реализации перед написанием кода. Агент исследует кодовую базу, задаёт уточняющие вопросы и генерирует проверяемый план. Нажми `Shift+Tab` в поле ввода, чтобы переключиться в режим планирования. Cursor автоматически предлагает этот режим при вводе ключевых слов, указывающих на сложность задачи.

Как работает:
1. Агент задаёт уточняющие вопросы для понимания требований.
2. Исследует кодовую базу для сбора контекста.
3. Создаёт проверяемый план реализации.
4. Редактируй план через чат или markdown файлы.
5. Нажми `Build` для выполнения плана.
Планы открываются как эфемерные виртуальные файлы. Чтобы сохранить план в проекте, используй `Save to workspace` — файл сохранится в `.cursor/plans/`.

# Видимость
To-do отображаются прямо в интерфейсе чата. Если настроена интеграция со `Slack`, список задач виден и там. Полный разбор задач доступен в любое время через интерфейс.

# Использование очереди
1. Введи следующую инструкцию во время выполнения текущей задачи.
2. Нажми `Ctrl+Enter`, чтобы добавить сообщение в очередь.
3. Сообщения появляются ниже активной задачи в порядке очереди.
4. Меняй порядок, кликая по стрелкам.
5. Агент обрабатывает их последовательно после завершения текущей работы.

# Переопределение очереди
Чтобы поставить сообщение в очередь вместо стандартной отправки, используй `Ctrl+Enter`. Чтобы отправить сообщение немедленно, в обход очереди, нажми `Cmd+Enter`. Такая «форс-отправка» выталкивает сообщение на выполнение сразу же.
