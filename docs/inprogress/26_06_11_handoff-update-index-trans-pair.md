## Что уже известно

### Схема trans-pair (эталон)

- Файл: `.agents/skills/other/translate/trans-pair.md`.
- Блок пары: строка `-`, строка RU, строка `-`, три строки `-`, строка EN.
- На входе — русские блоки; английский вставляется в пустой слот под русским; пустых строк в готовом файле быть не должно.

### Логика `update_index.py` (текущая, по коду)

- Точка входа: `update_index_html` → `_build_items` → для каждого `sources/**/*.txt` вызывается `file_passes_bilingual_pattern`.
- Не прошедшие попадают в `excluded` → `_update_gitignore` полностью перезаписывает `sources/.gitignore` (по одному пути на строку, без комментариев).
- Прошедшие — в `groups` → ссылки в `Pages/<папка>.html` и `index.html`.

### Проверка `file_passes_bilingual_pattern` (строки ~40–188)

1. Читает файл, режет на сегменты через `_split_by_dash_separators` (серии строк из одних `-` — разделитель).
2. `_drop_isolated_empty_segments` — убирает **один** пустой сегмент **между двумя непустыми** (лишняя пустая строка в разметке).
3. `_segments_pass_trans_pair_schema`:
   - `seg[0]` пустой;
   - все `seg[1..]` непустые;
   - нечётные индексы — язык `ru`, чётные — `en` (`_detect_block_lang`: сравнение числа кириллицы и латиницы);
   - последний индекс чётный (файл заканчивается EN-блоком полной пары).
4. Если «как есть» не прошло — в памяти вызывается `rebuild_trans_pair_lines` (сбор пар через `_extract_ru_en_pairs`, разбор склеек ` - - ` и ` - ` между RU/EN); если `rebuild` пустой → **отказ**; иначе снова `_lines_pass_trans_pair_schema`.
5. `rebuild_trans_pair_lines` **не записывает** файл на диск автоматически при прогоне `update_index` — только проверка в памяти.

### Вспомогательные функции пересборки (строки ~45–131)

- `_extract_ru_en_pairs` — берёт только подряд идущие пары с `ru` + `en` после слияния одноязычных фрагментов.
- Файлы **только на русском** дают `pairs=0`, `rebuild` пустой → не проходят.

- `sources/13/aw.txt` — в содержимом только русский текст в блоках с `-`; в чате зафиксировано: `pairs=0`, чётные слоты с языком `ru`, проверка не проходит.

## Подозрительности и неуверенности

- `_detect_block_lang` — эвристика (кириллица vs латиница); смешанный или короткий текст может дать неверный язык (не проверялось на граничных случаях в этом чате).
- `file_passes_bilingual_pattern` может пропустить файл **без перезаписи на диск**, если сырой файл битый, но `rebuild_trans_pair_lines` в памяти собирает валидные пары (пример из чата: `S001` — struct raw с пустым сегментом, при этом `pairs=21`, `reb_lang=True`).
- Ранее в чате была логика `_drop_trailing_empty_segments`, позже убрана из цепочки в пользу строгой схемы; **неуверенно**, остались ли в репозитории файлы, которые проходили только за счёт хвостовой обрезки до отката.
- `12_couz/Союз трех.txt` в `Pages/12_couz.html` — пользователь в чате явно не обсуждал, проходит ли он текущую проверку (в списке Pages есть).
- Файлы `S021`–`S068` в gitignore: в чате считалось, что у многих нет готового EN; **не проверялось** в этом handoff построчно каждый файл после финальной версии скрипта.
- `54 files included` — сумма по всем группам `sources/`, не только `12_couz`; детальный список 54 файлов в handoff не снимался.
